# EDA
El objetivo de este repositorio es comprender cómo deben tratarse los datos antes de comenzar a extraer la información 
que necesitamos.

En primer lugar, analizamos el tipo de datos con los que trabajamos y la estructura que presenta cada dataset. Para ello, 
hemos seleccionado dos conjuntos de datos diferentes:

  -Restaurantes (Zomato)
  
  Elegimos este dataset porque permite aplicar diversas técnicas de limpieza y procesamiento aprendidas durante el módulo de EDA. 
  Presenta problemas como valores nulos, codificación incorrecta, duplicados o datos inconsistentes, lo que lo convierte en un buen ejemplo práctico.
  
  -Modelo predictivo Videojuegos
  
  Este dataset se utiliza principalmente para realizar visualizaciones y analizar el comportamiento de los datos mediante gráficos.
  Su estructura más limpia lo hace adecuado para practicar agregaciones, comparaciones y análisis descriptivo.
